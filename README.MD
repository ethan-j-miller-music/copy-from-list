# copy\_from\_list.py — 使用 `tree /F` 清单按原层级批量拷贝文件

> 把这份 README 当成“学习札记 + 实用手册”。它不仅讲怎么用脚本，还讲为什么这么设计、踩过什么坑、如何扩展到更大更复杂的案例。

---

## 0) 这个脚本解决了什么问题？

你手里只有一份 **Windows `tree /F` 的文本输出**（形如“│、├─、└─”的目录树）**或“一行一个绝对路径”的文件清单**，但只想把其中**指定后缀**的文件（默认 `pdf`）从**某个源根目录**复制到**目标根目录**，并且**保留原有的多级目录结构**。

过去常见的做法是把这类“树形文本”**先解析回路径**，这一步很容易踩坑（混合缩进、BOM、编码、重复项、空目录等）。这个脚本做的就是**稳妥解析 + 安全复制**。

---

## 1) 快速开始（Quickstart）

1. 在 Windows 里先生成清单（在源根目录下）：

   ```bat
   tree /F > move_list.txt
   ```
2. 运行脚本（示例：只拷贝 pdf；先演练）：

   ```bat
   python copy_from_list.py "G:\源根\PDF" "E:\目标根" move_list.txt --ext pdf --dry-run
   ```
3. 确认无误后去掉 `--dry-run` 真正复制：

   ```bat
   python copy_from_list.py "G:\源根\PDF" "E:\目标根" move_list.txt --ext pdf
   ```
4. 多种扩展名：

   ```bat
   python copy_from_list.py "G:\源根\PDF" "E:\目标根" move_list.txt --ext pdf --ext docx --ext xlsx
   ```

> 如果你直接拥有**绝对路径清单**（例如 `dir /s /b *.pdf > list.txt` 的输出），同样使用以上命令即可。
> 脚本会自动识别清单格式，绝对路径会在复制前被转换成相对 `source_root` 的层级。

> **参数说明**
>
> * `source_root`：当时 `tree /F` 的那个目录（脚本按它来还原相对路径）
> * `target_root`：复制到哪里
> * `list_file`：清单文件（`tree /F` 文本或绝对路径列表，如：`move_list.txt`）
> * `--ext`：允许多次指定（不带点），默认 `pdf`
> * `--dry-run`：演练模式，只打印将要执行的复制

## 2) 解析原理（为什么以前会错？这次怎么修）

### 2.1 `tree /F` 的“缩进陷阱”

* Windows 的 `tree /F` 会用 **竖线 `│` + 空格** 混合来表示缩进层级。
* 很多“解析失败”的根因，是**把“文件行”的层级也靠缩进估算**，而在“最后一根分支”处，`tree` 常常**不用竖线只用空格**，导致**文件被错误挂到浅层目录**。

### 2.2 本脚本的核心策略

* **目录行**（匹配 `├─` 或 `└─`）才使用缩进估算层级，并维护一个**路径栈 `stack`**：`stack[depth] = 该层目录名`。
* **文件行**不再凭缩进估层级，**直接挂到当前目录栈的最深层**：

  ```python
  parts = stack + [name]
  rel_path = Path(*parts)
  ```
* 这样就**不受混合缩进影响**，保证“文件归属目录”稳定正确。

> 你可以把这理解为“小型状态机”：
>
> * 读到目录 → 更新状态（栈）
> * 读到文件 → 按“当前状态”直接落位

## 3) 代码结构与关键点

### 3.1 正则与命名分组

* **目录行**：`DIR_RE = r'^(?P<indent>[│\s]*)(?:├─|└─)(?P<name>.+?)\s*$'`
  抓取前导缩进 `indent` 与目录名 `name`。
* **文件行**：`compile_file_regex(exts)`
  生成形如

  ```regex
  ^(?P<indent>[│\s]*)(?![├└]─)(?P<name>[^\\/:*?"<>|]+\.(?:pdf|docx|...))\s*$
  ```

  * 先匹配缩进（仅用于“识别”，不会用它算层级）
  * `(?![├└]─)` 负向前瞻：排除目录行
  * `name` 捕获“合法文件名 + 指定扩展名”

### 3.2 路径栈（Path Stack）

* 读取到**目录行**时：

  1. `depth_of(indent)` 粗估层级（把 `│` 与空格都当宽度，**每 2 个字符 ≈ 一层**，足够鲁棒）
  2. 截断/扩展 `stack` 到该层级，再 `append` 当前目录名
* 读取到**文件行**时：

  * **不**用缩进估层级 → 直接 `stack + [文件名]` 得到相对路径

### 3.3 编码与健壮性

* 依次尝试 `utf-8-sig` → `utf-8` → `mbcs` 打开清单文件，兼容 BOM、本地编码。
* 路径去重、缺失项统计、演练模式、明确的终态统计行：

  ```
  Done. Copied: X, Missing: Y, Skipped(dup): Z
  ```

### 3.4 复制策略

* `shutil.copy2`：保留时间戳与元数据（相当于 Robocopy 的 `/COPY:DAT`）。
* 逐文件创建父目录：`dst.parent.mkdir(parents=True, exist_ok=True)`。

## 5) 常见问题（FAQ）

**Q1：清单不是 `tree /F` 的格式可以吗？**
A：可以。从 v2 起脚本会自动识别“`tree /F` 文本”和“绝对路径列表”两种清单。
   只要绝对路径都位于 `source_root` 之下，就能正常复制；不在源根内的条目会被跳过并在日志中提示。

**Q2：为什么不用“缩进估层级 + 文件行”一起算？**
A：`tree` 的缩进在某些分支会用空格“占位”，**文件行**的缩进**不稳定**。我们只在**目录行**估层级，文件行**跟随当前目录栈**，才稳定。

**Q3：中文目录、极长路径可以吗？**
A：可以。Python `pathlib` 对 Unicode 友好。极长路径在 Windows 老版本上可能需要启用“长路径支持”（组策略/注册表），但一般问题不大。

**Q4：性能如何？**

* 脚本是**单线程**，对“海量小文件”会比 **Robocopy（/MT 多线程）** 慢。
* 如果同类任务很多，建议后续做“并发复制”“按目录分组”等优化（见 §8）。

## 6) 更简单的替代清单：**绝对路径列表**

如果你的目标是“机器友好”，可以直接生成一份**一行一个绝对路径**的清单：

```bat
rem 在源根目录下
dir /s /b *.pdf > list.txt
```

> **直接使用脚本即可**：`python copy_from_list.py "G:\源根" "E:\目标根" list.txt --ext pdf`
>
> * 脚本会检测到清单为“绝对路径模式”，并把每一行折算成相对 `source_root` 的路径后再执行复制。
> * 如果清单包含不在 `source_root` 下的路径，终端会打印 `[SKIP]` 提示。

相比 `tree /F`，绝对路径清单没有“缩进歧义”，更适合脚本化生成（如 `dir /s /b`、PowerShell、数据库导出等）。

1. 从 `move_list.txt` 里挑出 2～3 个多级目录样本行，手动构造一个**极小子集清单**（只留这几行）。
2. 用 `--dry-run` 跑，确认输出的 `src -> dst` 路径是否吻合。
3. 去掉 `--dry-run` 真正复制，对比源与目的文件的大小与时间戳（`copy2` 会保留元数据）。
4. 把清单里重复出现的同名条目加进去，确认 `Skipped(dup)` 会统计而不会重复复制。

## 8) 进阶优化（未来可以做的升级）

* **并发复制**：`ThreadPoolExecutor(max_workers=8~16)` 包裹 `copy2`（注意针对机械盘 & SSD 分别调优并发度）。
* **差异跳过**：复制前比较目标同名文件的 `stat().st_size` 与时间戳，若一致则跳过（类似 Robocopy `/XO`）。
* **按目录分组**：把同父目录的文件放在一起批量处理，降低目录创建与文件系统元数据开销。
* **日志与重试**：记录失败项清单，提供二次补拷入口。
* **Robocopy 混合方案**：脚本只负责把清单“按父目录分组”，实际数据复制调用 Robocopy（`/MT` 多线程、`/Z` 断点续传等）。

---

## 9) 复杂度与鲁棒性

* **时间复杂度**：解析 O(N)（N 为清单行数），复制受 I/O 瓶颈主导。
* **空间复杂度**：路径栈深度通常很小（十级以内）。
* **鲁棒性**：

  * 编码三连尝试（`utf-8-sig`/`utf-8`/`mbcs`）
  * 路径去重
  * 演练模式与终态统计
  * 忽略清单里的无关行（盘符标题、空白行、统计行等）

## 11) 设计决策备忘（写给未来的自己）

* **不要**用文件行的缩进去推断层级 —— 只信“目录行 → 更新路径栈”。
* 为了健壮性，`depth_of` 采用“`│` + 空格”合并计数，“每 2 个字符 ≈ 一层” 的**宽松估计**即可。
* 用命名分组的正则（`(?P<indent>...)`, `(?P<name>...)`）提高可读性。
* `copy2` 能保留时间戳属性，满足“归档”场景的一致性要求。
* 清单可来自别人，不可控，所以**编码容错必须做**。

* 目录行（示例）

  ```

  ```

  匹配到：


  匹配到：

  * `indent="│  │      "`（仅用于识别，不用于层级计算）
  * `name="12345.pdf"`

### TL;DR（一图流）

```
清单（tree /F）
   │
   ├─ 目录行？→ 用缩进粗估 depth → 维护路径栈 stack[depth]=dirname
   │
   └─ 文件行？→ 直接挂到“当前栈的最深层”：Path(*stack, filename)
                                  │
                              相对路径列表
                                  │
                       src_root / rel_path  →  dst_root / rel_path
                                  │
                             shutil.copy2
```
